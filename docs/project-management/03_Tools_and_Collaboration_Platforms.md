# Tools and Collaboration Platforms

Throughout the project, a range of tools and technologies was used to support planning, communication, documentation, development, testing, and database integration. These tools helped the team organise the work more effectively and maintain a clearer development process.

The application was mainly developed using Visual Studio, with the user interface and scheduling features implemented in C# using Windows Forms. This was an appropriate choice because it provided an integrated development environment for building, running, and debugging a desktop-based application. Visual Studio also made it easier to manage the project files, edit code efficiently, and test new changes during development.

For database implementation, SQL Server LocalDB was used to create and manage the schedule database. This allowed the team to store schedule records persistently rather than relying only on temporary in-memory data. It also supported the coursework requirement for reading from and writing to a sample SQL database.

For version control and backup, GitHub was used throughout the project. This helped the team keep track of changes, protect the code from accidental loss, and maintain a more structured workflow. Branches were used so that team members could work on separate parts of the project before merging changes into the main working branch. Regular commits also made it easier to monitor progress and identify contributions made by different members.

The report diagrams, including the use case diagram, activity diagram, and flowchart, were created using draw.io. This tool was useful because it allowed diagrams to be produced clearly and edited easily as the system design developed. The written report and supporting documentation were prepared using Microsoft Word and Google Docs. Google Docs was particularly useful because it allowed all team members to contribute more easily, make live changes, and review documentation collaboratively in real time, while Microsoft Word was used for the more formal formatting of the final report documents.

For testing, the project used MSTest to verify that the main parts of the system behaved correctly. This supported a more reliable development process by helping the team identify and correct errors during implementation.

Communication was maintained through regular discussions and online messaging. This made it easier to share updates, discuss technical issues, confirm task allocation, and review project progress. Overall, the use of development, documentation, testing, and communication tools together helped the team work in a more organised and collaborative way throughout the coursework period.
